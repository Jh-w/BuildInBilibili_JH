## 1. 架构概览

### 1.1 系统边界

```
┌─────────────────────────────────────────────────────────────┐
│                        用户端                                │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │ B站App内Web   │    │ 微信内置浏览器  │    │ 移动浏览器    │  │
│  └──────┬───────┘    └──────┬───────┘    └──────┬───────┘  │
│         │                   │                   │           │
│         └───────────────────┼───────────────────┘           │
│                             │ HTTPS                         │
└─────────────────────────────┼───────────────────────────────┘
                              │
┌─────────────────────────────┼───────────────────────────────┐
│                    边缘层 (CDN)                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              EdgeOne Pages / CDN                      │  │
│  │         (index.html + static assets)                  │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────┼───────────────────────────────┘
                              │
┌─────────────────────────────┼───────────────────────────────┐
│                    计算层 (SCF)                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              FastAPI Backend (SCF)                     │  │
│  │         /api/chat  ·  /api/chat/stream                │  │
│  └──────────────────────────┬───────────────────────────┘  │
└──────────────────────────────┼──────────────────────────────┘
                               │
┌──────────────────────────────┼──────────────────────────────┐
│                      外部服务                               │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              DeepSeek API (chat completions)           │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 架构原则

| 原则 | 说明 | 体现 |
|------|------|------|
| **极简部署** | 单人开发+运维，最小化基础设施 | 静态前端 + SCF后端 + 外部AI |
| **移动优先** | 90%+ 流量来自手机 | 响应式设计，4G网络优化 |
| **零状态** | 后端不存储用户数据 | 无数据库，无会话管理 |
| **安全第一** | API Key不暴露前端 | 后端代理所有AI调用 |
| **降级优雅** | AI不可用时产品仍可用 | 预设回复 + 友好错误提示 |

## 2. 技术选型

### 2.1 技术栈总览

| 层 | 技术 | 版本 | 选型理由 |
|----|------|------|----------|
| **前端** | React + TypeScript | 18.x | 组件化开发，与跑步人格测试项目复用知识 |
| **构建** | Vite | 5.x | 快速HMR，构建产物小 |
| **样式** | CSS Modules / Tailwind CSS | — | 轻量，不引入组件库 |
| **后端** | FastAPI (Python) | 0.111+ | 异步支持好，与现有main.py兼容 |
| **AI服务** | DeepSeek API (OpenAI兼容) | chat/completions | 性价比高，中文能力强 |
| **HTTP客户端** | httpx (异步) | 0.27+ | FastAPI异步生态标配 |
| **部署-前端** | EdgeOne Pages / 静态CDN | — | 腾讯云EdgeOne免费额度 |
| **部署-后端** | SCF (云函数) / 轻量服务器 | — | 按量付费，低流量低成本 |
| **域名** | 待定（可复用跑步人格测试域名子路径） | — | — |

### 2.2 为什么选 DeepSeek

| 对比维度 | DeepSeek | OpenAI | 其他国产模型 |
|---------|----------|--------|------------|
| 中文能力 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐~⭐⭐⭐⭐ |
| 性价比 | ⭐⭐⭐⭐⭐ (¥1/1M tokens) | ⭐⭐ | ⭐⭐⭐⭐ |
| 人格化能力 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| API稳定性 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 合规性 | ✅ 国内可访问 | ⚠️ 需代理 | ✅ |

**结论**：DeepSeek 是中文毒舌场景的最优解，成本可控，能力足够。

```
┌─────────┐    POST /api/chat     ┌─────────┐    chat.completions    ┌──────────┐

│ (PNG)    │
└─────────┘
```

### 3.4 流式输出 vs 一次性返回

| 模式 | 用户感知延迟 | 实现复杂度 | 移动端兼容 | PRD要求 |
|------|------------|-----------|-----------|---------|
| 一次性返回 | 3-8秒全等 | 低 ✅ | 完美 ✅ | P0 |
| SSE流式 | 首个token 1-2秒 | 中 | 需处理部分浏览器SSE不支持 | P1 |

**MVP决策**：P0先用一次性返回（简单可靠），P1再加流式输出。

## 4. 数据设计

### 4.1 数据存储策略

**MVP阶段：不引入数据库**。理由：
- 产品是单次查询+分享场景，不需要持久化历史
- 零状态 = 零数据泄露风险 = 零存储成本
- 如需P1对话历史，使用 localStorage 存储最近5条（纯前端）

### 4.2 分数线数据

| 方案 | 优点 | 缺点 | MVP选择 |
|------|------|------|---------|
| 嵌入 System Prompt | 零延迟，实现简单 | token消耗大，更新需重新部署 | ✅ MVP采用 |
| RAG检索 | 数据量不受限，更新方便 | 增加延迟+复杂度 | P2考虑 |
| 实时API查询 | 数据最新 | 依赖外部服务，成本高 | 不做 |

**MVP数据范围**：2025年全国各省份主要高校文理科录取分数线，约50KB文本嵌入System Prompt。

### 4.3 数据结构

```typescript
// 前端输入
interface ChatRequest {
  message: string;  // "广东 物理类 625 计算机"
}

// 后端响应
interface ChatResponse {
  reply: string;    // Markdown格式的三段式内容
  // 错误时：
  error?: string;
}

// 埋点事件
interface AnalyticsEvent {
  event_id: string;
  device_id: string;
  timestamp: number;
  params: Record<string, string | number>;
}
```

## 5. API设计

### 5.1 接口清单

| 接口 | 方法 | 路径 | 描述 | 优先级 |
|------|------|------|------|--------|
| 前端页面 | GET | `/` | 返回静态HTML | P0 |
| 毒舌对话 | POST | `/api/chat` | 一次性返回完整回复 | P0 |
| 流式对话 | POST | `/api/chat/stream` | SSE流式返回 | P1 |
| 健康检查 | GET | `/api/health` | 返回服务状态 | P0 |

### 5.2 接口规格

#### POST /api/chat

```
Request:
{
  "message": "广东 物理类 625 计算机"
}

Response (200):
{
  "reply": "😈 毒舌时间\n625分想冲华科计算机？..."
}

Response (Error):
{
  "error": "API timeout",
  "reply": "😅 毒舌AI今天嗓子哑了，请稍后再试..."
}
```

**超时**：请求端8秒，后端DeepSeek调用6秒超时。

#### GET /api/health

```
Response (200):
{
  "status": "ok",
  "model": "deepseek-chat",
  "timestamp": "2026-06-23T08:00:00Z"
}
```

## 6. 安全设计

### 6.1 API Key管理

```
┌──────────────────────────────────────────────┐
│  前端 (公开)         后端 (私有)       外部   │
│                     ┌──────────┐             │
│  用户输入 ──────────→│ FastAPI   │─────→      │
│                     │          │ DeepSeek    │
│                     │ .env     │ API         │
│  用户 ←─────────────│ DEEPSEEK │←─────      │
│  (只看回复)         │ _API_KEY │             │
│                     └──────────┘             │
│                                              │
│  ❌ 前端永远不持有API Key                      │
│  ❌ 前端永远不直接调用DeepSeek                 │
└──────────────────────────────────────────────┘
```

### 6.2 安全措施清单

| 措施 | 实现 | 优先级 |
|------|------|--------|
| API Key 服务端持有 | .env文件，不提交git | P0 |
| CORS限制 | 仅允许产品域名 | P0 |
| 请求频率限制 | 同IP 10次/分钟（防止滥用） | P0 |
| 输入长度限制 | 最大500字符 | P0 |
| 内容安全 | System Prompt内置红线 | P0 |
| HTTPS | CDN + SCF均HTTPS | P0 |

## 7. 部署方案

### 7.1 推荐部署架构

```
用户 → DNS → EdgeOne CDN → 静态前端
                  │
                  └→ SCF (云函数) → DeepSeek API
```

### 7.2 替代方案（超轻量）

如果云函数配置复杂，可用轻量服务器直接部署：

| 方案 | 成本 | 运维 | 推荐度 |
|------|------|------|--------|
| **EdgeOne Pages + SCF** | ~0（免费额度内） | 低 | ⭐⭐⭐⭐⭐ |
| 轻量服务器（2C4G） | ~¥50/月 | 中 | ⭐⭐⭐ |
| Vercel + 后端转发 | ~0 | 低 | ⭐⭐⭐ (国内访问可能慢) |

### 7.3 部署检查清单

- [ ] 域名解析到EdgeOne
- [ ] SCF函数部署，环境变量配置DEEPSEEK_API_KEY
- [ ] 前端构建产物上传到EdgeOne Pages
- [ ] API路径代理配置（`/api/*` → SCF）
- [ ] HTTPS证书自动续期
- [ ] 健康检查接口验证
- [ ] 限流规则配置

## 8. 性能优化

### 8.1 优化目标

| 指标 | 目标值 | 优先级 |
|------|--------|--------|
| FCP (首屏渲染) | <1.5s | P0 |
| LCP (最大内容绘制) | <2.5s | P0 |
| TTI (可交互时间) | <3s | P0 |
| AI响应时间 (p95) | <8s | P0 |
| 分享卡片生成 | <2s | P0 |

### 8.2 优化策略

| 策略 | 实现 |
|------|------|
| 前端构建产物gzip/brotli压缩 | Vite插件自动处理 |
| 静态资源CDN缓存 | EdgeOne默认支持 |
| API响应缓存（同输入复用） | 可选：5分钟内相同输入返回缓存 |
| 骨架屏/loading状态 | 减少感知等待时间 |
| 图片懒加载 | 非首屏图片 |
| 代码分割 | React.lazy + Suspense |

## 9. 监控与可观测性

### 9.1 MVP监控方案

| 监控项 | 方案 | 备注 |
|--------|------|------|
| 前端埋点 | 自定义上报 → 自建/第三方 | 事件清单见PRD §13 |
| 后端日志 | SCF日志 / print输出 | 记录每次API调用耗时 |
| 错误监控 | try-catch + 错误事件上报 | API故障第一时间发现 |
| 性能监控 | Web Vitals上报 | FCP/LCP/TTI |

### 9.2 告警规则

| 告警 | 条件 | 通知方式 |
|------|------|---------|
| API错误率 > 5% | 10分钟内 | B站私信/飞书群 |
| AI响应 > 8秒 | 连续3次 | B站私信/飞书群 |
| 页面无法访问 | 健康检查失败 | B站私信/飞书群 |
