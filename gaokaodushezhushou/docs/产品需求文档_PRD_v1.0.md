## 1. 产品概述

### 1.1 产品定位（一句话）

> **「高考毒舌助手」是一款以「AI毒舌人格」为传播核心的轻量级对话产品，让考生在笑声中认清分数现实、获得靠谱志愿建议——顺便成为主合集的流量发动机。**

### 1.2 核心价值主张

| 利益相关方 | 一句话价值 |
|------------|-----------|
| **高考考生** | 输入分数，AI先毒舌再给数据——笑完发现自己更清醒了 |
| **B站观众** | 围观AI吐槽分数=天然短视频素材，比自己查志愿好玩十倍 |
| **主合集（参赛作品）** | 高考窗口期截流13天高密度流量，全部导入主合集 |

**Slogan**：「别的AI帮你算分，我帮你认清自己——顺便让你笑出声。」

### 1.3 产品角色

| 维度 | 定位 |
|------|------|
| 参赛角色 | 🔄 引流工具（不创建独立活动合集） |
| 产品形态 | 轻量Web对话页（无需安装、扫码即用） |
| 生命周期 | 短窗口高爆发（6/23-7/5核心窗口，可延伸至8/20） |
| 变现模式 | MVP阶段免费，价值体现在引流而非收入 |

## 2. 目标用户

### 2.1 用户画像

| 维度 | 🟢 高分段考生 | 🟡 中分段考生 | 🔴 低分段考生 |
|------|-------------|-------------|-------------|
| 分数段 | ≥本科线+30分（≥480分） | 本科线±30分（400-480分） | <本科线-30分（<400分） |
| 人群占比 | ~30% | ~40% | ~30% |
| 核心特征 | 有选择权，需要精准对标冲更高 | 焦虑选校，怕滑档，纠结「值不值」 | 不知还有什么路，需要鼓励+实际路径 |
| 痛点 | 冲稳保数据不够直观，容易被「差不多」建议糊弄 | 选择太多/太少，决策瘫痪 | 自尊心脆弱，需要被认真对待 |
| 使用动机 | 验证自己的判断，顺便看AI怎么怼自己 | 寻求第三方视角打破决策僵局 | 找到出路，证明「不止一条路」 |
| 毒舌耐受度 | 高（愿意被调侃） | 中（适度即可） | 零（禁用毒舌） |

**核心目标（Tier 1）**：🟢 高分段考生 —— 毒舌传播的主力人群，愿意截图分享「AI说我XXX」

### 2.2 核心痛点与产品解法

| 类型 | 痛点 | 产品解法 |
|------|------|----------|
| 🔴 | 考生对自己分数的竞争力缺乏现实认知 | 毒舌暴击+数据佐证 = 用幽默打破认知幻觉 |
| 🔴 | 信息过载：3000所学校1600个专业不知道怎么选 | 简化输入→给出3个层级（冲/稳/保），一次只解决一个问题 |
| 🟡 | 大厂工具太「冷」：数据正确但毫无情绪价值 | AI人设提供情绪价值——被怼完了反而开心 |
| 🟡 | 专业名字看不懂 | 专业翻译官：官方定义+真实生活+毕业去向 |
| 🟢 | 查完志愿无处分享/表达 | 一键生成截图卡片，天然社交货币 |

### 2.3 分享动机模型

| 层级 | 动机 | 占比估算 | 设计启示 |
|------|------|----------|----------|
| 🥇 自我表达 | 「AI说我625不配华科，你们评评理」 | ~50% | 截图卡片需包含完整的毒舌+数据对比 |
| 🥈 社交货币 | 「这个东西太搞笑了，快试试你的分数」 | ~30% | 分享卡片自带产品入口链接 |
| 🥉 求助 | 「帮我看看这个冲稳保靠谱吗」 | ~20% | 卡片保留数据部分完整可读 |

### 2.4 核心使用场景

| 场景 | 描述 | 传播机制 |
|------|------|---------|
| **出分日首测** | 考生查到分的第一时间，来「挨骂」+ 拿建议 | 情绪峰值期，截图分享意愿最强 |
| **志愿纠结期** | 在两个学校之间摇摆，来问AI对比 | 反复使用，每次可能分享不同结果 |
| **围观吃瓜** | 没高考/已毕业的人围观，输入自己的「当年分数」 | 非目标用户但贡献传播量 |
| **专业选择** | 不知道某个专业是干嘛的，来问翻译官 | 长尾搜索场景 |

## 3. 用户旅程

### 3.1 核心旅程地图（高分段用户·出分日首测）

```
┌──────────────────────────────────────────────────────────────────┐
│ 阶段     [发现]  →   [输入]  →   [等待]  →   [收获]  →   [传播]  │
├──────────────────────────────────────────────────────────────────┤
│ 用户心理  好奇+紧张  认真+期待   期待+不安   被怼→笑→清醒  爽+想分享  │
│ 触点     B站视频    Web页面    AI生成中   聊天结果页   分享卡片   │
│ 时长     —          15s        3-8s        30-60s      5s       │
│ 流失风险  没点链接   表单太长    响应太慢    毒舌太过火   无分享按钮  │
│           |           |           |           |           |      │
│ 情绪     😏        🤔        ⏳        😂→🤔→😏   📸       │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 关键步骤详细描述

#### 步骤0：发现入口

| 触点 | 描述 |
|------|------|
| **B站视频结尾CTA** | 视频结尾：「带着你的分数来挨骂 👉 链接在评论区」 |
| **B站动态** | 出分日当天的动态：「出分了？来，输入你的分数」 |
| **评论区互动** | UP主在热门评论下回复带链接 |

**关键要求**：
- 链接必须零门槛（无需登录、无需下载App）
- 移动端优先（考生90%用手机查分）
- 加载时间 < 2秒（焦虑期用户耐心极低）

#### 步骤1：输入页

**设计要点**：
- 极简：只需输入「省份 + 选科 + 分数 + 意向方向」4个字段
- 输入框带示例 placeholder 降低认知负担
- 「心情选择」可选开关（P1功能，MVP可降级为固定模式）
- 提交按钮文案有态度：「来，让我看看你几斤几两 😏」

#### 步骤2：等待/AI生成中

**交互规则**：
- 打字机效果逐字输出（非一次性返回，增加戏剧感）
- 显示「毒舌AI正在组织语言...」loading状态
- 最长等待8秒（超时则降级返回完整结果）

#### 步骤3：结果页

**三段式输出**（高分段完整版）：

```
┌─────────────────────────────────────┐
│ 😈 毒舌时间                          │
│ （1-3句精准打击分数认知）              │
│                                     │
│ 📊 现实一点                          │
│ 🔴 冲刺：院校·专业（参考分）          │
│ 🟡 稳妥：院校·专业（参考分）          │
│ 🟢 保底：院校·专业（参考分）          │
│                                     │
│ 🤖 AI自首                           │
│ （自嘲式免责声明）                    │
│                                     │
│ ═══════════════════════════════════ │
│ [📸 生成分享卡片]  [🔄 再问一次]      │
└─────────────────────────────────────┘
```

#### 步骤4：分享传播

**分享卡片设计要素**：
- 顶部：用户输入摘要（「广东·物理类·625分·计算机」）
- 中间：毒舌金句（最扎心的一句）
- 底部：产品名称 + 二维码/链接
- 配色：深色背景 + 毒舌风格（非工具感）

## 4. 核心内容设计

### 4.1 System Prompt 人设设计

#### 人格设定

| 维度 | 设定 |
|------|------|
| **名字** | 毒舌AI |
| **语气** | 像朋友吐槽，不像教导主任 |
| **节奏** | 快、不废话、每句话有信息量 |
| **底线** | 吐槽分数不吐槽人、有数据才说、结束自嘲 |

#### 分层输出规则

| 分数段 | 毒舌力度 | 输出格式 | 首句 |
|--------|---------|---------|------|
| 🔴 <400分 | **0%（全面关闭）** | 跳过毒舌段，直接给路径建议 | 「高考分数不定义你的人生。」 |
| 🟡 400-480分 | 40%（最多1句轻量吐槽） | 吐槽融入数据段，不单独成段 | 正常开场，语气收敛 |
| 🟢 ≥480分 | 100%（毒舌拉满） | 完整三段式：毒舌→数据→自首 | 该怼就怼 |

#### 毒舌内容红线（硬约束）

> 以下话题**绝对禁止**任何形式的毒舌、讽刺或负面评价：

| 禁止话题 | 触发时的处理 |
|---------|------------|
| 家庭经济状况 | |
| 身体健康/残疾 | |
| 地域歧视 | 不做任何评价 |
| 性别 | → |
| 学校层次羞辱（如「二本出来送外卖」） | 「这个问题我不方便评价， |
| 考生外貌/年龄/任何个人特征 | 我们来聊聊你的志愿吧。」 |

### 4.2 专业翻译官模式

当用户输入非分数格式（如「生物医学工程怎么样」），自动切换：

```
📖 官方说法：（一句话）
🫠 真实生活：（毒舌版，有信息量）
💼 毕业去向：（真实可信）
🤖 AI自首：「以上是一个AI的理解，去找学长学姐聊聊比听我的靠谱。」
```

## 6. 功能需求

> P0 = MVP 必须上线。P1 = MVP 验证后迭代。P2 = 未来探索。P3 = 长期储备。

### 6.1 P0 功能（MVP必须）

| 编号 | 功能 | 描述 | 用户故事 | 验收标准 |
|------|------|------|----------|----------|
| **P0-1** | 志愿毒舌评估 | 用户输入省份+选科+分数+方向，AI返回三段式毒舌诊断 | 作为考生，我想输入分数后看到AI的毒舌点评+真实数据，以便清醒认识自己分数能去哪 | ① 输入4字段后3-8秒内返回结果 ② 结果包含毒舌段+数据段+自首段 ③ 低分自动切换安慰模式 |
| **P0-2** | 分数分档自适应 | AI根据用户分数自动选择毒舌力度和输出格式 | 作为低分考生，我不想被嘲讽，需要实际的出路建议 | ① <400分毒舌关闭，首句=鼓励 ② 400-480分毒舌收敛 ③ ≥480分毒舌拉满 |
| **P0-3** | 内容红线防护 | AI绝对不在敏感话题上毒舌 | 作为产品方，我需要确保AI不对经济/健康/性别/地域/学校层次进行负面评价 | 触发敏感话题时回复「这个问题我不方便评价…」 |
| **P0-4** | 分享卡片生成 | 用户可一键生成包含毒舌金句+数据的截图卡片 | 作为高分考生，我想把我被AI怼的结果截图发朋友圈/B站动态 | ① Canvas生成640×1008px图片 ② 包含毒舌金句+冲稳保数据+产品名+入口链接 ③ 生成时间<2秒 |
| **P0-5** | 专业翻译官 | 用户输入专业名称，返回官方定义+真实生活+毕业去向 | 作为考生，我想知道「生物医学工程」到底学什么、出来能干嘛 | ① 自动识别非分数输入 ② 返回三部分：定义/真实生活/去向 ③ 结尾自嘲免责 |
| **P0-6** | 移动端适配 | 页面在手机端完美显示和交互 | 作为考生，我用手机查分后直接打开链接使用 | ① iOS Safari/Chrome + Android Chrome 正常显示 ② 输入表单不溢出 ③ 分享按钮可触发原生分享 |

### 6.2 P1 功能（MVP验证后迭代）

| 编号 | 功能 | 描述 | 用户故事 | 优先级依据 |
|------|------|------|----------|------------|
| **P1-1** | 心情选择开关 | 用户进入前可选「毒舌拉满/轻度毒舌/别怼了」 | 作为用户，我今天心情不太好，不想被怼太狠 | 舆情风险报告中最高优先级的防护措施 |
| **P1-2** | 流式输出（打字机效果） | AI回复逐字显示而非一次性返回 | 作为用户，我想看AI一句一句怼我，更有戏剧感 | 提升用户停留时长和传播意愿 |
| **P1-3** | 对话历史（会话内） | 保留当前会话最近5条对话 | 作为用户，我想追问「那西电呢？」而不重新输入所有信息 | 提升连续使用体验 |
| **P1-4** | 诊断卡视觉增强 | 添加诊断戳、案号、法院印章等视觉元素 | 作为传播受众，看到截图觉得有设计感才会转发 | 提升分享卡片点击率和二次传播率 |
| **P1-5** | 分享卡片扫码入口 | 分享卡片底部二维码可扫码直接进入产品 | 作为看到朋友截图的人，我想扫码自己也试试 | 形成完整的传播闭环 |

## 7. 非功能需求

> 留待架构总监细化。仅标注产品侧约束。

| 类别 | 需求 | 目标值 | 依据 |
|------|------|--------|------|
| 性能 | 首屏加载时间 | <2秒（4G网络） | 焦虑期用户耐心极低 |
| 性能 | AI响应时间 | <8秒（p95） | 超过8秒用户可能放弃等待 |
| 并发 | 峰值并发 | 支持100 QPS | 出分日瞬时流量冲击 |
| 兼容性 | 移动端浏览器 | iOS 14+ Safari, Android 8+ Chrome | 考生主要用手机 |
| 安全 | API Key保护 | 不暴露在前端代码中 | DeepSeek API Key安全 |
| 安全 | 内容过滤 | System prompt内置硬约束 | 防止毒性输出 |
| 可用性 | 零登录 | 不需要任何注册/授权 | 最大化转化率 |
| 可用性 | 错误降级 | API失败时返回友好错误提示 | 不让用户看到技术报错 |

## 8. 成功指标

### 8.1 北极星指标

> **K因子 > 1.2**（每个用户平均带来≥1.2个新用户，形成正向传播飞轮）

**计算方法**：K = 分享卡片生成次数 × 卡片打开率 × 打开后使用率

**目标值**：K > 1.2（Week 1），K > 1.5（Peak）

### 8.2 核心 KPI

| 漏斗层级 | 指标 | 计算方式 | 目标值（Week 1） | 备注 |
|----------|------|----------|-----------------|------|
| 触达 | 视频播放量 | B站后台 | 50,000+ | EP0+出分视频合计 |
| 点击 | 链接点击率 | 点击/播放 | >3% | 评论区置顶+视频结尾CTA |
| 使用 | 产品UV | 埋点上报 | 3,000+ | 唯一设备ID |
| 使用 | 毒舌评估完成率 | 完成评估/进入页面 | >60% | 表单填写不流失 |
| 传播 | 分享卡片生成率 | 生成分享/完成评估 | >25% | 核心传播指标 |
| 传播 | K因子 | 新用户/分享带来的用户 | >1.2 | 北极星指标 |
| 引流 | 主合集点击率 | 点击主合集链接/UV | >5% | 引流目标 |

### 8.3 Go/No-Go 决策节点

```
        内容验证（6/15前）      内测验证（6/20）       全量发布（6/23）      持续运营（7/5）
        ─────────────────      ────────────────      ────────────────      ────────────────
        毒舌质量≥3.5/5          NPS>0                  K因子>0.8            K因子>1.2
        GO/NO-GO                GO/NO-GO              GO/NO-GO              GO/NO-GO
              │                     │                     │                     │
              ▼                     ▼                     ▼                     ▼
        进入开发              发布到小范围            全量发布             继续运营/归档
                             B站动态先行             出分日同步
```

| 决策节点 | 条件 | GO | NO-GO | 灰区 |
|----------|------|-----|-------|------|
| **内容验证** | 招募≥5人试测，毒舌质量中位数≥3.5/5 | 进入开发 | 重写system prompt+扩大试测，不启动工程 | — |
| **内测验证** | 小范围发布（B站动态），收集20+使用反馈，NPS>0 | 6/23全量发布 | 调整毒舌力度/修复问题，延迟发布 | NPS -10~0：追加优化后Go |

## 9. 商业模式

### 9.1 当前阶段（MVP）

| 角色 | 商业价值 | 衡量方式 |
|------|---------|----------|
| **引流引擎** | 将高考流量导入B站主合集 | 主合集点击量 |

| **社交货币** | 截图分享降低获客成本 | K因子 |

### 9.2 阶段变现策略

```
MVP阶段（6/23-7/5）：    免费，纯引流。价值体现在流量导入而非收入。
P2阶段（7/5-8/20）：    可能植入高考/大学相关品牌广告（需UV>10万）。
P3阶段（2027+）：       IP化运作，可独立商业化（付费深度分析/IP授权）。
```

**结论**：MVP不追求直接变现。商业成功的定义 = 「为主合集带来多少投币/关注」。

## 13. 埋点需求

### 13.1 埋点事件清单

| 事件 ID | 事件名称 | 触发时机 | 上报参数 | 用途 |
|---------|----------|----------|----------|------|
| `page_view` | 页面浏览 | 用户打开产品页面 | device_id, referrer, screen_size | 计算UV+来源渠道 |
| `form_submit` | 提交评估 | 用户点击提交按钮 | device_id, province, score_range, subject | 计算转化率+用户画像 |
| `result_view` | 查看结果 | AI返回结果渲染完成 | device_id, duration_ms, score_range, mode | 计算完成率+性能监控 |
| `share_generate` | 生成分享卡片 | 用户点击分享按钮 | device_id, score_range, template | 计算分享率（核心KPI） |
| `share_complete` | 分享卡片生成成功 | Canvas渲染完成 | device_id, generate_ms | 分享漏斗+性能监控 |
| `share_click` | 分享卡片被点击 | 他人通过分享卡片链接进入 | device_id, sharer_device_id | K因子分子 |
| `translate_submit` | 使用专业翻译 | 用户输入非分数查询 | device_id, query_type | 专业翻译官使用量 |
| `cta_click` | 点击主合集链接 | 用户点击引流CTA | device_id, cta_position | 引流转化率 |
| `error_occur` | 错误发生 | API失败/超时/异常 | device_id, error_type, error_msg | 监控产品稳定性 |

### 13.2 K因子追踪方案

```
K因子 = 新用户（通过分享进入）/ 分享者数量

追踪链路：
  用户A 生成分享卡片 → share_generate (记录 sharer_device_id)
  → 用户B 通过卡片链接进入 → page_view (referrer=share, 附带 sharer_device_id)
  → K因子分子 +1
```

## 14. 风险与假设

### 14.1 关键假设

| 编号 | 假设 | 验证方式 | 不成立时的应对 |
|------|------|----------|---------------|
| **H1** | 高考考生愿意被AI「毒舌」，且毒舌后不感到被冒犯 | 内容验证（≥5人试测，≥3.5/5分） | 降低全局毒舌力度 → 改为可选开关 → 如仍无效则转为温和调侃 |
| **H2** | 毒舌内容有传播性，用户愿意截图分享 | 内测20人数据，分享率>15% | 优化分享卡片设计 → 加强毒舌金句的可截图性 → 如仍无效则重新评估产品方向 |
| **H3** | 静态数据集（2025年）足够支撑MVP阶段的志愿建议 | 抽样对比2025 vs 2026数据偏差<5% | 如偏差大则紧急接入2026数据 |
| **H4** | DeepSeek API在出分日不会限流/故障 | 提前压测+准备backup prompt | 降级为静态回复+缓存常见分数段结果 |
| **H5** | B站评论区链接可正常跳转Web页面（不折叠/屏蔽） | EP0视频发布后验证 | 改用动态文字引导+搜索关键词 |
| **H6** | 13天窗口够完成MVP-反馈-迭代循环 | 内测6/20前完成 | 砍P1功能，集中资源保P0 |
