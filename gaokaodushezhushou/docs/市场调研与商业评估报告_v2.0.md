## 1. 摘要

### 核心结论

**「AI 高考志愿 × 人格化娱乐内容」是一个已被验证的蓝海，且已实现 MVP 交付。短期目标（引爆 6/23-7/5 高考窗口）已完成，当前进入复盘与长尾期评估。**

### 关键发现（5 条）

| # | 发现 | 置信度 |
|---|------|--------|
| 1 | **蓝海已验证**：大厂全员下场做工具，但零竞品做人格化——毒舌助手是唯一人格化 AI 高考产品 | ✅ 高 |
| 2 | **窗口期判断准确**：13 天核心窗口（6/23-7/5）流量密度极高，产品已在出分日上线，抓住了窗口 | ✅ 高 |
| 3 | **安全机制充分**：四道防线（吐槽分数不吐槽人+毒舌后给数据+AI自首+低分段保护）覆盖主要舆情风险 | ✅ 高 |
| 4 | **产品轻量但完整**：29 项开发任务全部交付，Vanilla JS 单文件 SPA + FastAPI + DeepSeek，约 640 行前端 + 187 行后端 | ✅ 已交付 |
| 5 | **下一步关键**：上线 7 天实际数据复盘 → 决定是否扩展为「大学新生全周期产品」→ 持续引流主合集 | ⚠️ 待验证 |

### Go/No-Go 判断

| 阶段 | 判断 | 依据 |
|------|------|------|
| MVP 上线（6/23） | ✅ **已 GO，已交付** | 窗口不可错过，成本可控 |
| 扩展 V2.0 | ⚠️ **待上线数据验证** | 需看 7 日 UV、分享率、导流效果 |
| 资产化 | 🔮 **远期** | 毒舌人设积累后可考虑 IP 化 |

## 2. 调研目标

**一句话**：评估「AI 高考毒舌助手」作为 B 站内容 IP 引流工具的市场可行性——从上线前预期验证升级为上线后复盘与扩展评估。

**具体问题**：
1. 高考志愿 AI 市场是否如预期（1290 万考生 + 大厂围剿）？
2. 「人格化娱乐」差异化是否扛住了真实用户检验？
3. 13 天窗口期后的长尾策略是否可行？
4. 毒舌安全边界在真实运行中是否稳固？
5. 商业模式从「免费引流」升级的空间在哪里？

```
2023 ── ChatGPT 出圈，AI 教育话题萌芽

2026 ── 全员下场年 ← 我们在这里
        ├─ 6/7-9 考试：大厂 AI 工具主动锁死功能（防作弊）
        ├─ 6/23 出分日：全年流量峰值
        ├─ 6/23-7/5：13 天志愿填报窗口
        └─ 趋势：AI 从「辅助工具」向「决策 Agent」进化

2027+ 预测：
  → AI 志愿填报成为标配（像今天的天气预报 app）
  → 差异化从「功能准确度」转向「情感价值 + 信任」
  → 人格化 AI 将从「新奇」变为「刚需」
```

## 5. 用户洞察

### 5.1 分层用户画像

#### 🎯 主要用户：高三刚出分的考生（P0）

| 属性 | 描述 |
|------|------|
| 年龄 | 16-19 岁 |
| 人口基数 | 1290 万（2026 高考报名） |
| B 站渗透 | 估算 60-70% 的 18 岁群体使用 B 站 |
| 设备 | 手机为主（90%），PC 为辅（10%） |
| 核心场景 | 6/23 出分后，用手机查「XXX 分能上什么大学」 |
| 情绪状态 | 焦虑（不知道去哪） + 期待（终于出分了） + 迷茫（选什么专业） |
| 决策窗口 | 7-14 天（各省批次截止日） |
| 信息渠道 | B站（视频）+ 百度（搜索）+ 微信（家长转发）+ 同学群 |

#### 🔄 次要用户：大学生/过来人（P1）

| 属性 | 描述 |
|------|------|
| 年龄 | 19-23 岁 |
| 核心场景 | 刷 B 站推荐流看到 → 怀旧共鸣 → 转发给学弟学妹 |

#### 👀 围观用户：职场新人/路人（P2）

| 属性 | 描述 |
|------|------|
| 年龄 | 23-28 岁 |
| 核心场景 | 纯娱乐——「这 AI 好损，看看怎么说别人的分数」 |
| 价值 | 播放量贡献 + 话题扩散（微博/朋友圈） |

#### ⚠️ 敏感用户：家长群体（P2）

| 属性 | 描述 |
|------|------|
| 年龄 | 40-55 岁 |
| 核心场景 | 帮孩子参谋志愿 → 验证 AI 推荐是否靠谱 |

### 5.2 痛点与痒点

| 层级 | 痛点 | 痒点（传播触发） | 毒舌助手解法 |
|------|------|-----------------|------------|
| 🔴 P0 | 出分后不知道能去哪 | 想跟人吐槽自己的分数 | 毒舌暴击→数据→自嘲，一站式 |
| 🔴 P0 | 3000 所学校 1600 个专业不知道怎么选 | 想看到「别人什么分去了哪」 | 冲稳保三段式数据 |
| 🟡 P1 | 专业名字看不懂 | 想听「真实的专业生活」 | 专业翻译官：毒舌版 |
| 🟡 P1 | 父母/二舅乱建议 | 想截图发朋友圈「AI 说我这个分…」 | Canvas 分享卡片 |
| 🟢 P2 | 出分后焦虑没处释放 | 想被逗笑 | 毒舌吐槽本身就是情绪出口 |

### 5.3 分享动机模型

```
用户分享毒舌截图的心理路径：

1️⃣ 自嘲 → 「AI 说我 625 分连门卫微信都加不上 😂」
   动机：用幽默降低自己的焦虑，博朋友一笑

2️⃣ 求证 → 「AI 说我这分段可以冲 XX，你们觉得呢？」
   动机：把决策焦虑分散到社交圈，寻求社会验证

3️⃣ 炫耀 → 「AI 说我这分数还不错，至少没说想冲清华」
   动机：高分考生用自嘲方式做软性炫耀

4️⃣ 共情 → 「当年我填志愿的时候要是有人骂我就好了」
   动机：过来人的怀旧共鸣，转发给学弟学妹
```

**K 因子预估**：1.5-3.0（每个用户平均带来 1.5-3 个新用户）

### 5.4 核心使用场景

| 场景 | 触发时机 | 用户行为 | 情绪弧线 |
|------|---------|---------|---------|
| **出分日首查** | 6/23 查完分后 1 小时内 | 输入分数 → 被毒舌 → 看到数据 → 截图发群 | 😰焦虑 → 😂笑 → 🤔思考 → 📤分享 |

| **围观吃瓜** | 任何时间 | 不输入分数，看 quick chips 示例、刷视频 | 😂纯娱乐 |
| **过来人共鸣** | 录取季 | 输入已录取学校，「当年这分现在能去哪」 | 🥲怀旧 → 😂共鸣 → 📤转发 |

## 6. 产品设计分析

> ⚠️ 以下为市场视角的功能优先级与交互建议，**仅供产品总监参考，非决策指令**。

### 6.1 从传播漏斗反推产品设计

```
传播漏斗                         产品设计应对
──────────────────────────────────────────────────
触达（视频播放 50-100万）  ──→  EP0 预告 + 出分日视频 + B站搜索SEO
  ↓ 3-5% 点击率
访问（产品 UV 3-5万）     ──→  移动端极致适配，零注册摩擦
  ↓ 80% 完成一次对话
使用（2.4-4万）            ──→  三段式输出确保用户拿到价值
  ↓ 15% 分享
传播（截图卡片 3600-6000） ──→  Canvas 诊断卡是核心传播载体
  ↓ K因子 1.5-3.0
二次触达（5.4-18万）      ──→  卡片上的链接/二维码导回产品
```

### 6.2 功能优先级判断（市场视角）

| 功能 | 当前状态 | 对传播的贡献 | 对留存的贡献 | 建议优先级 |
|------|---------|------------|------------|-----------|
| 毒舌志愿评估 | ✅ P0 已交付 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 维持 |
| 专业翻译官 | ✅ P1 已交付 | ⭐⭐⭐ | ⭐⭐⭐ | 维持 |

### 6.4 「心情选择器」的 Go/No-Go 判断

```
论据支持：
  ✅ 降低脆弱用户的毒舌恐惧
  ✅ 把「被怼」变成「主动选择」——降低举报动机
  ✅ 提供用户掌控感

  ❌ 增加一步交互 → 降低完成率
  ❌ 分段规则已覆盖大部分场景（高分段自动毒舌、低分段自动关闭）
  ❌ 可能稀释「毒舌」的核心人设

建议：P2 保持，先看 V1.0 上线数据。
如果低分段投诉率 > 1%，加速上线心情选择器。
```
