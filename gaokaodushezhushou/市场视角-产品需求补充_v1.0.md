## 二、需要补充的维度

### 2.1 上线后数据复盘机制（缺失）

PRD 当前没有定义「上线后如何评估产品效果」。从市场视角，以下数据必须在 7/5 窗口结束前获取：

| 数据 | 采集方式 | 为什么重要 | 当前状态 |
|------|---------|-----------|---------|
| 产品 UV（独立访客） | 前端埋点 PV/UV | 验证 SOM 预估 | 🔴 **未接入** |
| 分享卡片生成率 | Canvas `toBlob()` 事件 | 验证传播假设 K因子 | 🔴 **未接入** |
| 视频 → 产品转化率 | UTM 参数 / Referrer | 验证引流链路效率 | 🔴 **未接入** |
| 低分段/中分段/高分段使用占比 | System Prompt 输出格式可区分 | 验证用户分层假设 | ⚠️ 需手动抽样 |
| DeepSeek API 调用总量 | API 后台统计 | 反推 UV 下限 | ⚠️ 可查 |

**建议**：PRD 补充 §8 成功指标中增加「数据采集方式」列，标注每个指标的当前接入状态。

### 2.2 K 因子验证（缺失）

PRD §2.3 有分享动机模型，但缺少 K 因子（病毒系数）的目标值和验证方式。

```
当前 PRD 有：分享动机三层
缺少的：每个用户平均带来多少新用户？如何追踪？

建议补充：
  K因子目标：1.5-3.0
  验证方式：
    - 短链点击统计（如果分享卡片嵌入链接）
    - 社交平台品牌搜索量（搜索「毒舌AI」「高考毒舌助手」）
    - 评论区「从哪来的」归因
```

### 2.3 Phase 2 扩展的产品决策节点

PRD §9 版本规划有 V2.0「大学新生毒舌助手」，但缺少明确的 **Go/No-Go 决策标准**。

建议补充：

| 决策节点 | 时间 | GO 条件 | NO-GO 条件 |
|---------|------|---------|-----------|
| Phase 2 启动 | 7/5 | UV > 2 万 + 分享率 > 10% + 无舆情事故 | UV < 5000 或出现舆情 |
| 内容持续投入 | 7/25 起每 2 周 | 单视频播放 > 5000 | 连续 3 期 < 3000 |

### 2.4 心情选择器的优先级论证

PRD §3.5 将心情选择器定为 P2。市场视角的补充论证：

**赞成的数据**：
- 降低脆弱用户摩擦 → 提升低分段使用率
- 把「被怼」变成「主动选择」→ 降低举报动机

**反对的数据**：
- 多一步交互 → 转化漏斗多一层流失（行业均值：每多一步 -15-25%）
- 当前分段规则已覆盖 90% 场景

**建议**：维持 P2，但设定触发条件——如果上线数据显示低分段用户跳出率 > 30%，则提升至 P1。

## 三、产品与市场的接口问题

### 3.1 引流效率追踪

产品是「主合集的流量发动机」。但当前 PRD 没有定义发动机的「马力表」：

| 缺少的仪表盘 | 为什么需要 |
|------------|-----------|
| 产品 UV → 主合集点击率 | 验证引流 ROAS（内容资产回报） |
| 分享卡片 → 二次访问占比 | 验证传播飞轮是否转动 |
| 各渠道来源占比 | 优化投放策略 |

**建议**：PRD 补充「引流仪表盘」章节。

### 3.2 毒舌人格浓度监控

AI 人格是核心竞争力，但「浓度」可能随时间漂移（temperature 调节、模型更新等）：

**建议**：PRD 补充「AI 人格浓度」验收标准——每日抽查 10 条随机回复，人工评分毒舌味 1-5 分，低于 3 分触发 System Prompt 调优。
