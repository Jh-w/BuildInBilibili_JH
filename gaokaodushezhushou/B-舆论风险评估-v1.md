# B 高考毒舌助手 · 舆论风险评估

> 评估日期：2026-06-09
> 问题：毒舌会不会伤考生？会不会被举报？

---

## 一、风险是真实的

| 风险 | 触发场景 | 严重程度 |
|------|---------|---------|
| 考生情感伤害 | 出分日本来就焦虑，被AI一顿怼，心态崩了 | ⚠️ 高 |
| 家长举报 | 「你们这个AI怎么能这么跟孩子说话」 | ⚠️ 中 |
| B站社区举报 | 「恶意博眼球」「不尊重考生」 | ⚠️ 中 |
| 比赛取消资格 | 规则明确：「明显偏离赛事宗旨或存在恶意博眼球行为」 | 🔴 高 |
| 舆情反噬 | 如果有人截毒舌截图发微博「AI在高考出分日嘲讽考生」 | 🔴 高 |

---

## 二、但你方案里已经有防护措施了

### 现有的四道防线

```
防线1: 吐槽分数，不吐槽人
  ✅ 「625分连门卫微信都加不上」← 吐槽的是分数和现实差距
  ❌ 「你这种人也配学计算机？」← 攻击人，绝对不能出现

防线2: 毒舌之后立刻给实际帮助
  ✅ 三句毒舌 → 📊 冲稳保数据分析 → 用户得到了真实价值
  ❌ 只毒舌不给方案 = 纯恶意

防线3: AI自首（自降权威）
  ✅ 「以上都是AI说的，不一定对，但比二舅靠谱」
  ❌ 摆出权威姿态：「本AI判断绝对正确」

防线4: 免责声明
  ✅ 「最终决定问你自己和你爸妈」
```

### 为什么这四道防线有效

用户心理路径：
```
被怼了 😤 → 拿到有用数据 🤔 → AI自己认怂 😂 → 没那么生气了
```

关键不是「不毒舌」，而是**毒舌之后必须给糖**。

---

## 三、还需要加强的地方

### 3.1 情绪安全阀：最低分段保护

**问题场景**：一个考了300分的考生输入分数，被说「你这分数蓝翔都够呛」。

→ 这不是毒舌，是残忍。

**建议**：在 system prompt 里加一条「低分段软化」规则：

```
如果用户分数明显偏低（低于本科线）：
- 毒舌力度自动降低 70%
- 吐槽不超过一句
- 立刻切换到「现实路径建议」模式
- 语气从「你怎么这么菜」变成「路不止高考一条」
```

### 3.2 产品前端加「心情选择」

用户进入产品前先选心情：

```
┌──────────────────────────┐
│ 今天想被怎么对待？        │
│                          │
│ 😈 毒舌拉满 — 往死里怼    │
│ 😐 轻度毒舌 — 温柔一点    │
│ 🫂 别怼了 — 直接给数据    │
└──────────────────────────┘
```

把选择权交给用户。选了「毒舌拉满」的人被怼了不会举报——他们自己选的。

### 3.3 视频内容的安全边界

| 可以做 | 不可以做 |
|--------|---------|
| 用自己的分数当例子 | 用真实观众的真实低分当众处刑 |
| 「625分想冲华科？睡醒没？」 | 「350分的人别浪费教育资源了」 |
| 展示产品功能 | 标题党「AI说你不配上大学」 |
| 毒舌+数据+自嘲三段式 | 纯毒舌没有后文 |
| 自嘲「AI说的不一定对」 | 摆出「本AI的判断就是真理」 |

### 3.4 视频标题和封面的自我审查

```
❌ 危险标题：
  「你的分数就是垃圾——AI毒舌志愿助手」
  「300分的人别填志愿了」
  「AI说你这分数只配去专科」

✅ 安全标题：
  「别的AI帮你算分，我帮你认清自己」
  「AI说我的分数连门卫微信都加不上（但后面的话很有用）」
  「被AI骂了3分钟之后，我填的志愿反而更清醒了」
```

## 四、核心判断

### 「毒舌」会不会被骂，取决于「毒舌完之后给了什么」

| 毒舌完给的是 | 用户感受 | 举报风险 |
|------------|---------|---------|
| 什么都没有 | 😡 纯恶意 | 🔴 高 |
| 一堆冰冷数据 | 😐 工具而已 | 🟡 中 |
| 有用数据 + AI自己认怂 + 给后路 | 😂 这人有点意思 | 🟢 低 |

### 我们走第三条路。

**毒舌不是目的，是让用户在笑的同时接受现实的手段。**

---

## 五、一句话总结

> **毒舌是钩子，数据是价值，认怂是安全阀。三样缺一不可。只要毒舌之后用户拿到了真实有用的信息 + AI自己先认输，举报动机就不成立。**

但如果有人断章取义只截毒舌部分——那是舆情风险。所以每条视频的标题和封面必须包含「数据/帮助」的信号，不能只强调毒舌。
