## 版本命名规则

```
<文件名>-v<N>.<ext>

示例：
  产品方案-A-身体翻译官-v1.md    → 初版
  产品方案-A-身体翻译官-v2.md    → 大改（方案架构调整）
  B-EP0-预告视频脚本-v1.md       → 初版
  B-EP0-预告视频脚本-v2.md       → 大改（内容重大调整）

小修小改（错别字、格式微调）不升版本号，直接覆盖 + Git commit
```

---

## Git 规范

```
Commit message 格式：
  <类型>: <简短描述>

类型：
  init     — 新建文件
  update   — 内容更新（不改版本号）
  bump     — 版本号递增（重大修改）
  rename   — 文件重命名
  fix      — 修复错误
  style    — 格式/样式调整
```
