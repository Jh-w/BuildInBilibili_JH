# 🏃 C 选项：跑步人格测试 2.0 — 完整产品方案

> 定位：从「做题测人格」升级为「AI 跟你聊出来的跑步人格」
> 参赛角色：🏆 主合集备选（最早可上线）

---

## 一、产品一句话

**「不用做题。告诉AI你最近跑了什么步、买了什么装备、跟谁一起跑的——让它吐槽出你都不知道的跑步人格。」**

---

## 二、v1 → v2：核心变化

| | v1（已上线） | v2 |
|---|---|---|
| 交互方式 | 8道选择题 → 等结果 | 自由对话 → AI 动态判断 |
| 人格输出 | 固定16型 | 16型 + 动态微调 + 个性化吐槽 |
| 数据采集 | 用户手动选 | 对话中自然获取 |
| 分享力 | Canvas卡片 | 对话截图 + Canvas卡片（双模式） |
| AI介入 | 无（纯规则引擎） | DeepSeek 驱动全流程 |
| 用户感知 | 「我做了一个测试」 | 「AI 看穿了我的跑步人格」 |

---

## 三、核心交互流程

```
用户进入产品
  → AI开场：「嗨，我是跑步人格侦探。别紧张，不做题——跟我聊聊你最近跑步的事。」
  → 对话式采集（3-5轮）：
      AI：「你最近一次跑步是什么时候？跑了多远？」
      用户：「上周六，跑了8公里。」
      AI：「是自己跑的还是有跑友？跑完发了朋友圈吗？」
      用户：「一个人跑的，没发。」
      AI：「那天穿的什么鞋？是新买的还是穿了好久的？」
      用户：「穿了半年的亚瑟士，底都快磨平了。」
  → AI 分析 + 人格输出：
      「检测完毕。你是『城市流浪跑者 🗺️』——独自跑、不炫耀、鞋磨平了还在穿。
       你不是在跑步，你是在用脚步丈量孤独。帅。」
  → 分享卡片生成（Canvas）
```

## 四、技术架构

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | React + TypeScript + Vite | 对话界面（复刻微信聊天） |
| AI | DeepSeek API | system prompt = 跑步人格侦探 |
| 逻辑层 | 16型人格框架（保留v1体系） | AI 对话后将结果映射到人格体系 |
| 部署 | EdgeOne Pages + SCF | 跟v1同架构，可快速复用 |
| 分享 | Canvas 分享卡片 + 对话截图 | 双模式分享 |

### System Prompt 核心设计

```
你是「跑步人格侦探」，你的任务：
- 通过自然对话了解用户的跑步习惯
- 3-5轮对话内提取关键信息：训练频次、装备偏好、社交习惯、计划风格
- 将用户匹配到16型跑步人格之一
- 用毒舌但有爱的语气告知结果

对话节奏：
- 第1轮：了解最近跑步情况（频次/距离）
- 第2轮：了解社交行为（独跑/团跑/发朋友圈/参加比赛）
- 第3轮：了解装备态度（实用主义/装备党/随缘）
- 第4轮（如需要）：了解训练计划（严格跟计划/随心跑/佛系）

禁止：
- 不要问超过5轮
- 不要像在做问卷调查
- 不要透露评分逻辑
```

## 九、如果 C 当主合集

**优势**：最早发内容，抢跑领先；已有v1用户基础可导流。

**劣势**：概念新鲜度不如A；评委可能认为「这是在已有产品上加AI」而非「AI创造的新产品」。

**补救方案**：强化「AI创造」属性——
- v1是规则引擎，v2是AI原生
- 对话体验是v1完全不具备的
- 可以强调「AI如何从对话中自己判断人格」的技术故事
