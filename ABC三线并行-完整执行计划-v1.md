### 引流链路设计

```
B 高考引流视频（高考搜索流量 + 话题流量）
  → 视频内容：毒舌AI吐槽志愿/专业
  → 结尾 CTA：
      「想被AI再吐槽一次？来测测你的跑步人格 👉 [C链接]」
      「顺便看看我给器官做的聊天App，心脏会骂人那种 👉 [A链接]」

A 参赛视频（B站活动流量 + 内容推荐流量）
  → 视频内容：器官拟人化AI产品展示
  → 结尾 CTA：
      「喜欢这个？投个币支持一下 🙏」
      「跑者福利：来测测你的跑步人格 👉 [C链接]」
```

### 为什么 A+B 都导流给 C？

1. **C 是已有产品（v1已上线）**，有用户基础和内容话题
2. **跑步垂直社群忠诚度高**，引流转化好
3. **C 的产品形态（人格测试）是天然流量入口**——用户测完会分享，分享带来新用户
4. **A 的投币和 C 的用户增长不矛盾**——观众可以既投币 A 又测 C
5. **比赛结束后 A 的流量会衰减**，但 C 作为独立站外产品可以持续运营

### 5.3 B 不参赛的设计原则

```
✅ B 产品本身不创建活动合集
    → 高考窗口结束后自然沉寂，不拖累比赛节奏

✅ B 的开发投入最小化
    → 核心功能极简：输入分数→毒舌输出→分享卡片
    → 不追求完整度，追求传播力
```
