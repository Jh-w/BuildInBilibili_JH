# ABC 三线并行策略 — 比赛最优解

> 核心逻辑：比赛考核投稿数+投币数。三个产品 = 三倍投稿量 + 三个不同受众群 = 最大化双指标。

---

## 一、能不能同时做？能。关键在错峰。

| 产品 | 开发量 | 开发窗口 | 内容生命周期 | 角色定位 |
|------|--------|---------|-------------|---------|
| **B 高考毒舌** | ⭐ 最小（API wrapper + 前端壳） | 6/9-6/15 | 6/16-7/5（短爆） | 闪电战，抢投稿数 |
| **C 跑步2.0** | ⭐⭐ 已有代码，只加AI对话 | 6/9-6/16 | 6/17-8/20（长尾） | 先发优势，最早投稿 |
| **A 身体翻译官** | ⭐⭐⭐ 最大（从零开发） | 6/9-7/1 | 7/2-8/20（长尾） | 主力产品，冲投币数 |

**错峰逻辑**：C 最先上线（复用代码）→ B 蹭高考窗口 → A 接棒跑后半程。三个产品不在同一周开发，不打架。

---

## 二、10 周作战时间线

```
Week 1  (6/9-15)  🔨 开发 C + B
                    🎬 C EP0：「我把跑步人格测试加了AI，它开始毒舌我了」

Week 2  (6/16-22) 🎬 C EP1（人格分类揭秘）
                    🎬 B EP0 预热：「高考出分前夜，我让AI预测你的志愿」
                    🔨 B 上线

Week 3  (6/23-29) 🎬 B EP1-2（高考出分周，毒舌点评志愿，流量高峰）
                    🎬 C EP2（AI对话效果精选）
                    🔨 开始开发 A（身体翻译官）

Week 4  (6/30-7/6) 🎬 C EP3（跑步人格深度解读）
                    🎬 B EP3（高考志愿吐槽合集，B收官）
                    🔨 A 开发中

Week 5  (7/7-13)   🎬 C EP4
                    🔨 A 收尾，EP0 预告

Week 6  (7/14-20)  🎬 A EP0：「我把你的器官变成了人，然后他们开始吵架」
                    🎬 C EP5

Week 7  (7/21-27)  🎬 A EP1（心脏+肺）
                    🎬 C EP6

Week 8  (7/28-8/3) 🎬 A EP2（胃+肝）
                    🎬 C EP7

Week 9  (8/4-10)   🎬 A EP3（大脑+肌肉）
                    🎬 C EP8

Week 10 (8/11-17)  🎬 A EP4（肾脏+皮肤，A系列收官）
                    🎬 C EP9（完结篇：16人格全复盘）
```

**投稿总数**：
- C 跑步2.0：EP0-EP9 = **10 期**
- B 高考毒舌：EP0-EP3 = **4 期**
- A 身体翻译官：EP0-EP4 = **5 期**
- **合计：19 期投稿** ← 单产品策略的 2-3 倍

## 三、每周投入估算

| 周 | 开发任务 | 视频制作 | 总投入/周 |
|----|---------|---------|-----------|
| W1 | C开发（复用代码，快）+ B开发（API壳） | 1 期 | 高 |
| W2 | B收尾 | 2 期 | 中 |
| W3 | A开发启动 | 2 期 | 中高 |
| W4 | A开发中 | 2 期 | 中 |
| W5 | A开发收尾 | 1 期 | 中 |
| W6-10 | 仅维护 | 1-2 期/周 | 低 |

前三周最忙（开发 C + B + 启动 A），之后匀速。可承受。

---

## 四、为什么三线并行比单产品强

| | 单产品（A 或 C） | ABC 三线 |
|---|---|---|
| 总投稿数 | 8-10 期 | **19 期** |
| 覆盖受众 | 1 个圈层 | 3 个圈层（高考生/跑步/泛科普） |
| 6 月有内容吗 | ❌（A 开发中） | ✅（C+B 已经发了） |
| 7 月中-8 月中 | ✅ | ✅（A+C 双线） |
| 8/20 数据累积 | 单线累积 | **三线累积** |
| 投币理由多样性 | 单一 | 三重（你帮了我志愿 / 你懂跑步 / 你科普好有趣） |

## 五、风险 & 对策

| 风险 | 对策 |
|------|------|
| 前三周太忙 | C 的核心代码已有，B 是极简 prompt wrapper，真正从零开发只有 A |
| B 生命周期太短 | 它就是闪电战用的，4 期够产生 6 月投稿数。投币靠 A 和 C |
| 内容质量下降 | 每期视频控制在 3-5 分钟，不追求长视频。Build in Public 的特点就是 raw and real |
| A 开发延期 | C 已经有 10 期内容保底，A 延期 1-2 周不影响比赛覆盖率 |

---

## 六、一句话总结

> **C 先跑保底投稿数，B 蹭高考短爆窗口，A 接棒冲投币质量。三线错峰，覆盖 73 天全时段。** 

单产品最多 10 期投稿，三线并行能做到 19 期——在考核投稿数的比赛里，这是降维打击。
